```text
请参考我上传的两张图片，为小学 1–6 年级儿童阅读产品设计“完成单个学习子任务”的轻量奖励反馈界面。竖屏移动端画幅 393×852，不要手机外壳，不要做成大弹窗，不遮挡主要学习内容，不打断学习节奏。
```

## 生成顺序建议

如果不准备一次生成全部界面，建议按以下顺序：

1. 今日计划完成结算弹窗。
2. 家具面板。
3. 兑换确认弹窗与星星不足弹窗。
4. 兑换成功和摆放成功状态。
5. 家具、装饰图鉴。
6. 单个子任务反馈和通用轻提示组件。

最先生成家具面板的原因是它能确定后续卡片、星星价格、已拥有、使用中和弹窗的共同视觉语言。
