# 游戏音效接入设计

## 目标

根据 `D:\YDXDR_Game\sounds\audio-mapping.md`，把正式音频素材接入当前静态 HTML/CSS/JavaScript 游戏流程。音效必须在对应交互发生时播放，背景音乐必须在对应页面循环，并且同一次交互只触发一个匹配音效。

## 素材映射

正式接入以下五个文件，并复制到 `assets/audio/`，使用英文运行时文件名：

| 来源文件 | 运行时文件 | 用途 | 循环 |
| --- | --- | --- | --- |
| `入口进入1.mp3` | `entrance-enter.mp3` | 首页点击地图入口 | 否 |
| `地图页2.mp3` | `map-bgm.mp3` | 地图页背景音乐 | 是 |
| `房间内.mp3` | `reading-hut-bgm.mp3` | 阅读小屋背景音乐 | 是 |
| `点击1.mp3` | `hut-item-click.mp3` | 小屋家具/装饰面板及物品素材点击 | 否 |
| `点击2.mp3` | `ui-click.mp3` | 其他可操作按钮和链接 | 否 |

`入口进入2.mp3` 和 `地图页1.mp3` 是备用素材，本次不复制、不加载。

## 触发规则

### 首页

- `.hero-ip` 点击后立即触发 `entrance-enter`，确保在 0.1 秒内开始播放。
- 首页其他实际可操作的按钮或链接触发 `ui-click`。
- 首页现有地图转场及导航时序保持不变。

### 地图页

- 页面加载后尝试循环播放 `map-bgm`。
- 返回首页、进入阅读小屋、进入成就页等可操作链接触发 `ui-click`。

### 阅读小屋

- 页面加载后尝试循环播放 `reading-hut-bgm`。
- 家具入口、装饰入口、面板关闭，以及家具/装饰物品卡片点击触发 `hut-item-click`。
- 返回、图鉴入口、兑换弹窗按钮等其余可操作按钮和链接触发 `ui-click`。
- 物品拖动后的抑制点击不触发音效，因为该交互未形成有效点击。

### 图鉴和成就页

- 返回链接、图鉴标签等实际可操作按钮和链接触发 `ui-click`。

## 音频架构

新增独立的 `audio-manager.js`，负责：

- 保存固定音频清单和页面背景音乐映射。
- 预加载并播放一次性音效。
- 通过事件委托识别普通按钮、普通链接和带特殊音效标记的元素。
- 特殊音效优先于通用 `ui-click`，避免一次点击重复播放。
- 对快速连续点击允许每次都重新播放对应音效。
- 在地图页和阅读小屋启动循环背景音乐。
- 捕获浏览器自动播放拒绝；若背景音乐无法自动开始，则在首次 `pointerdown` 或键盘交互时重试。
- 音频加载或播放失败时静默降级，不阻止按钮行为、动画或页面导航。

页面通过语义化 `data-audio` 标记声明特殊音效和背景音乐。未标记的真实 `button`、带 `href` 的 `a` 元素由统一事件委托播放 `ui-click`。音频使用素材原始音量，本次不新增音量、静音或设置界面。

## 页面导航

普通链接保持浏览器原生导航，不为完整播放点击音效增加等待时间。首页入口已有约 1.5 秒转场，因此入口音效可在转场期间播放。页面离开时，其页面背景音乐随文档销毁自然停止。

## 测试策略

本功能改变点击和页面加载行为，采用 RED → GREEN → REFACTOR：

1. 单元测试固定音频清单、循环设置、特殊音效优先级、通用点击匹配和自动播放重试行为。
2. 结构测试确认所有页面加载统一管理器、正式素材存在、备用素材未进入运行时目录，以及特殊交互具有正确标记。
3. 浏览器测试使用可观察的音频替身确认首页入口、小屋面板、小屋物品、普通按钮和背景音乐分别触发正确音频且不重复。
4. 最后运行项目完整 Node 测试与现有 Playwright 回归测试。

## 验收标准

- 五个正式音频文件位于 `assets/audio/` 并被正确引用。
- 首页入口在点击后 0.1 秒内触发指定入口音效。
- 地图页和阅读小屋分别尝试循环播放指定背景音乐，并可在自动播放受限后通过首次交互恢复。
- 小屋家具/装饰面板和物品素材使用 `点击1`。
- 其他真实按钮和链接使用 `点击2`。
- 单次点击不会重复触发特殊音效与通用点击音效。
- 音频失败不会破坏现有页面行为。
- 备用音频不进入运行时资源。
- 现有项目回归测试通过。
