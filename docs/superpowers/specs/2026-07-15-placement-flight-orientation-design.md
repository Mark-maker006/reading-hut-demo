# 阅读小屋放置飞行动效设计

## 目标

为家具和装饰面板中的全部 7 个可放置物品增加统一、丝滑的飞行旋转与镜像过渡。飞行素材可以从 Panel 中的任意朝向起飞，但抵达幽灵位时必须与房间中最终成品层的位置、尺寸和方向一致，不得在烟雾或落位回弹阶段突然跳变。

## 适用范围

- 阅读地毯
- 木地板
- 小书架
- 小绿植
- 星星墙贴
- 雪梨玩偶
- 故事背景墙

不修改兑换、星币、背包排序、图鉴或放置状态的业务规则。

## 动效方案

采用目标驱动动画。动画运行时读取 Panel 卡片缩略图的当前朝向，并读取房间最终成品层的实际几何与镜像状态，而不是把幽灵位外框直接当作成品尺寸。

飞行阶段约为 1100ms：

1. 素材从卡片当前可见方向和尺寸起飞，避免首帧翻面。
2. 沿现有贝塞尔曲线路径上扬，执行一次连续的 Y 轴 3D 翻转。
3. 同时加入幅度较小的 Z 轴摆动，形成自然的抛物线姿态变化。
4. 飞行过程中持续插值尺寸、位置和朝向。
5. 抵达时与最终成品层的中心、宽高和镜像方向一致。
6. 命中后继续使用现有烟雾、闪光和落位回弹效果。

当起点和终点方向相同，Y 轴完成一整圈翻转并回到原方向；当方向不同，Y 轴完成半圈翻转并停在目标方向。最终方向由目标成品层决定，不由 Panel 卡片数据硬编码。

## 目标几何解析

飞行动效接收两个目标：

- `targetElement`：幽灵位容器，负责命中状态、烟雾中心和最终放置状态。
- `targetVisualElement`：容器中的 `.placement-item`，负责提供最终位置、尺寸和方向。

隐藏状态下的 `.placement-item` 仍保持可测量的绝对定位盒子，只通过 `visibility` 控制显示。这样飞行前即可获得它的真实矩形，不需要临时修改 DOM 或维护重复坐标。

### 木地板特殊情况

木地板没有独立的房间成品图。放置后仅隐藏灰色虚线覆盖层，露出背景图自带的正常地板。

因此木地板飞行时：

- 目标中心使用地板幽灵位的中心。
- 目标尺寸保持卡片素材的正常展示尺寸，不扩展到 393×511 的地板外框。
- 命中后按现有逻辑隐藏虚线层，不创建彩色房间图片层。

## 朝向解析

新增可测试的朝向与关键帧计算逻辑：

- 识别 `transform: matrix(...)`、`matrix3d(...)` 和独立 `scale` 属性中的水平镜像。
- 将起点和终点转换为 Y 轴角度。
- 起点角度与 Panel 缩略图当前可见方向一致。
- 终点角度与目标成品层方向一致。
- 关键帧使用 `perspective(...) rotateY(...) rotateZ(...)`，并保留现有位移和缩放。

故事背景墙的目标成品层保持现有 `scale: -1 1`，飞行动画会自动读取该状态，不再需要在动画代码中写死物品 ID。

## 动画降级与清理

- 当用户启用 `prefers-reduced-motion`，直接触发命中和放置，不执行飞行、旋转或镜像动画。
- 动画成功、失败或被中断时，都必须移除临时飞行图片、烟雾节点和路径状态。
- 放置状态仍由现有 `ReadingHutState.placeItem()` 统一提交。

## 测试

使用 RED → GREEN → REFACTOR：

1. 普通物品：验证相同起止方向时完成整圈翻转，终点尺寸与成品层一致。
2. 镜像物品：验证起点朝向、途中 Y 轴角度和终点镜像方向连续变化。
3. 木地板：验证没有成品层时不会缩放到幽灵位的大尺寸。
4. 页面接线：验证 `targetElement` 和 `targetVisualElement` 被分别传入动效函数。
5. 回归：运行完整 Node 测试套件，确认兑换、卡片排序、放置状态和其他物品布局不变。
