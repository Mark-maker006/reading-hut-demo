# 首页到地图云层转场设计

## 目标

点击首页 IP 后，用 IP 作为视觉中心，将页面文字、学习卡片和底部导航在约 400ms 内向四周径向推开；随后蓝湖云层从屏幕四周进入，在遮住 IP 后显露地图终版，并在约 820ms 时无跳帧进入 `level-map.html`。

## 交互时序

- `0–400ms`：昵称、本周学习、今日阅读、接下来计划和底部导航沿远离 IP 的方向散开，同时轻微淡出、缩小和模糊。
- `180–350ms`：IP 保持原位，轻微提亮，继续作为视觉焦点。
- `220–700ms`：四组云层从上、下、左、右进入；地图终版在云层后方逐渐显现。
- `620–760ms`：云层遮住首页 IP，地图合成图中的 IP 在遮挡后接管画面。
- `700–820ms`：动画云层抵达并对齐地图最终云层位置，然后淡出，露出完全相同的地图合成图。
- `820ms`：写入一次性来源标记并导航到地图页。

## 结构

### 首页散开层

为 `.profile`、`.weekly`、`.reading-card`、`.next-plan` 和 `.tabbar` 增加统一的 `data-home-transition-piece` 标记。运行时读取每个元素与 `.hero-ip-art` 的中心点，通过归一化向量计算散开方向，避免写死响应式坐标。

### 固定转场舞台

首页增加一个固定定位、默认隐藏的 `.home-map-transition`：

- `.transition-map-preview` 使用现有 `assets/map-final-lanhu.png`。
- 四个 `.transition-cloud-piece` 使用同一张蓝湖云层 PNG，通过裁剪区域区分上、下、左、右云团。
- 云层起点位于屏幕外，终点为 PNG 的原始位置；抵达后淡出，使地图终版里已经烘焙的云层接管画面。

## 状态模型

状态顺序固定为：

```text
idle → scattering → cloud-covering → settling → navigating
```

- 首次点击后立即锁定，重复点击返回 `false`，不创建第二套计时器。
- 转场期间锁定滚动和其他指针操作。
- 导航前写入 `sessionStorage['home-map-transition-v1'] = 'pending'`。
- 地图页只消费一次该标记；直接打开或刷新地图页不播放转场。

## 动效参数

- 首页散开：`400ms cubic-bezier(0.22, 1, 0.36, 1)`。
- 云层进入：从 `220ms` 开始，持续约 `500ms`。
- 地图显现：在云层进入中后段用约 `180ms` 淡入。
- 云层交接：`100ms` 淡出。
- 散开距离根据视口较长边和元素尺寸计算，保证卡片离开当前视觉中心。

## 降级与异常

- `prefers-reduced-motion: reduce`：跳过大范围平移，使用约 `180ms` 淡出后导航。
- 若云层图未完成加载：继续执行散开并用白色雾层完成遮挡，不阻塞导航。
- 若 `sessionStorage` 不可用：导航仍正常，只跳过来源标记。

## 可访问性

- 保留原来的 `<a href="./level-map.html">`，键盘 Enter 与点击走同一控制器。
- 动画层 `aria-hidden="true"`。
- 动画期间 IP 链接设置 `aria-disabled="true"`，结束或初始化失败时恢复。

## 测试

交互行为使用 RED → GREEN：

- 验证径向向量始终远离 IP。
- 验证首次点击与重复点击保护。
- 验证状态在 `220ms`、`700ms`、`820ms` 的转换。
- 验证导航只在最终状态发生，并写入一次性标记。
- 验证 reduced-motion 在约 `180ms` 导航。
- 浏览器关键帧验证：首页初始、约 `200ms` 散开、约 `500ms` 云层进入和地图最终帧。

## 验收

- 首页散开阶段约 400ms，IP 不向外散开。
- 云层从四周进入并形成明显遮挡。
- 动画结束时云层位置与当前地图合成图一致。
- 页面切换没有白闪、重复动画或双重导航。
- 首页和地图页原有交互保持可用。
