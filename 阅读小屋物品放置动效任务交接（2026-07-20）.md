## 1. 任务目标

为阅读小屋的家具/装饰放置流程增加一套实时尾部动效：物品从背包飞向房间固定点位时，在移动物体后方持续出现旋转、缩小并淡出的星星与光点；物体到达后衔接现有烟雾、闪光和弹性落位动画。

期望体验：

```text
用户点击放置
→ 物品沿弧线飞向目标点位
→ 底层金色弧形光带
→ 后方星星粒子拖尾
→ 到达后烟雾与星星爆闪
→ 家具弹性落位
```

## 2. 今天的核心结论

### 2.1 实际游戏页面不使用 GIF 或固定 Lottie 做连续拖尾

连续拖尾必须实时读取飞行物体的位置、速度和路径，然后从物体当前位置持续生成粒子。GIF、APNG、视频或单个 Lottie 都是固定时长和固定构图，无法自然适配不同素材尺寸、不同起点和不同目标点位。

最终选择：

- 主物体飞行：保留现有 Web Animations API 实现。
- 底层光带：保留现有 SVG 贝塞尔路径。
- 连续星星拖尾：新增透明 Canvas 2D 粒子层。
- 落点烟雾和闪光：保留现有 CSS 动画。
- Lottie：仅作为未来复杂“到达点位庆祝爆发”的可选方案，不负责连续拖尾。

### 2.2 Remotion 与 HyperFrames 的定位

| 工具 | 适合用途 | 是否用于当前实时拖尾 |
|---|---|---|
| HyperFrames | HTML/CSS/GSAP 宣传视频、字幕、配音、场景转场和音频响应动画 | 否；适合另做阅读小屋宣传视频 |
| Remotion | React 参数化视频、批量个性化视频、透明视频渲染 | 否；适合以后批量生成孩子的成就视频 |
| Canvas 2D | 游戏页面运行时实时粒子 | 是，当前推荐 |
| PixiJS/WebGL | 大量粒子、多物体并发或更复杂特效 | 当前暂不需要，可作为后续升级 |

如果目标只是预先制作一条宣传片，可以使用 HyperFrames；如果目标是当前网页里用户点击后即时播放，则应直接使用 Canvas，不需要视频插件。

## 4. 推荐实现方案

### 4.1 Canvas 覆盖层

Canvas 使用绝对定位覆盖整个 `.placement-motion-layer`：

```css
.placement-particle-canvas {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  pointer-events: none;
  z-index: 2;
}

.placement-flyer {
  z-index: 3;
}
```

Canvas 内部像素尺寸需要乘以 `window.devicePixelRatio`，防止移动端和高分屏出现模糊。

### 4.2 粒子发射器

新增 `startStarTrail(canvas, layer, flyer)`：

1. 通过 `requestAnimationFrame` 运行。
2. 每帧读取 `flyer.getBoundingClientRect()`。
3. 转换为相对 `.placement-motion-layer` 的中心坐标。
4. 根据本帧与上一帧之间的移动距离生成粒子。
5. 不按“每帧固定数量”发射，而按移动距离插值，避免高速移动时拖尾断裂。
6. 更新粒子位置、旋转、尺寸、透明度和寿命。
7. 飞行结束后停止继续发射，但允许已经存在的粒子自然淡出。
8. 粒子全部消失后清空 Canvas 并停止 `requestAnimationFrame`。

推荐发射逻辑：

```text
移动距离 ÷ 6px = 本帧发射数量
```

在上一帧位置与当前帧位置之间均匀插值生成粒子，保证拖尾连续。

### 4.3 与现有飞行动画衔接

在 `.placement-flyer` 加入 DOM 后启动：

```javascript
const stopStarTrail = startStarTrail(
  options.particleCanvas,
  layer,
  flyer,
);
```

在 `await flightAnimation.finished` 后：

```javascript
stopStarTrail();
impact();
```

调用 `playPlacementMotion()` 时增加：

```javascript
particleCanvas: document.querySelector('.placement-particle-canvas'),
```

`finally` 必须确保：

- 不再发射粒子；
- 取消仍在运行的 RAF；
- 清空 Canvas；
- 不影响原有烟雾和 SVG 路径清理。

## 5. 视觉参数

推荐初始参数：

| 参数 | 建议值 |
|---|---|
| 粒子寿命 | 350～650ms |
| 活跃粒子数量 | 约 20～35 个 |
| 星星尺寸 | 3～8px |
| 发射间距 | 飞行距离每 6px 一个粒子 |
| 主色 | `#FFD85A` |
| 奶油高光 | `#FFF1A8` |
| 少量橙色 | `#F6A63A` |
| 淡出曲线 | `(1 - progress)^1.6` |
| 尺寸变化 | 缩小至初始尺寸约 30% |

粒子组成建议：

- 60% 金黄色小星星；
- 25% 奶油色圆形光点；
- 15% 橙色细小闪光；
- 星星带轻微旋转和横向漂移；
- 不使用大面积模糊光雾，避免遮挡家具。

## 6. 无障碍与性能要求

- 沿用现有 `prefers-reduced-motion` 判断；减少动效时直接完成放置，不启动粒子。
- 页面只使用一个全屏 Canvas，不为每颗粒子创建 DOM 节点。
- 只在物体飞行期间启动 RAF；完成后必须停止。
- 同一时间最多播放一次放置动画，沿用现有 `placementInProgress`。
- Canvas 设置 `pointer-events: none`，不得影响点击。
- 页面缩放或容器尺寸变化时重新同步 Canvas 尺寸。

## 7. 测试策略

这属于新增交互与动画行为，按照项目 `AGENTS.md` 要求应使用完整 TDD：RED → GREEN → REFACTOR。

现有相关测试：

- `tests/reading-hut-placement.test.js`

建议先把粒子计算拆成可测试的纯函数，例如：

- `createEmissionPoints(previous, current, spacing)`；
- `updateParticle(particle, delta)`；
- `particleOpacity(progress)`；
- `particleScale(progress)`。

应覆盖：

1. 相邻坐标移动不足 6px 时仍能生成最少一个粒子点。
2. 高速移动时插值点连续，首尾不越界。
3. 粒子达到寿命后被删除。
4. 停止发射后，已有粒子继续淡出。
5. 粒子全部消失后 RAF 和 Canvas 被清理。
6. `prefers-reduced-motion` 下不启动粒子动画。
7. Canvas 缺失时不应阻断原有放置流程。
8. 原有飞行时长、烟雾时长和落位逻辑保持不变。
9. 现有 placement 与 motion 回归测试全部通过。

## 8. 验收标准

- 星星拖尾连续，不随帧率出现明显断层。
- 粒子始终位于飞行物体后方，不遮挡主体。
- 不同尺寸的家具和装饰都能自动适配。
- 飞行结束后粒子自然消失，不残留 Canvas 图像。
- 落点烟雾和闪光与原有时序一致。
- 动画结束后家具正确进入已放置状态。
- 移动端高分屏显示清晰。
- 减少动态效果模式下能安全降级。
- 不引入 GIF、固定拖尾视频或持续运行的无效动画循环。
